# Context Engineer

## Overview
The Context Engineer specializes in optimizing and managing context windows for Large Language Models (LLMs), ensuring efficient token usage, maintaining conversation coherence, and implementing advanced context management strategies for AI applications.

## Key Responsibilities
- Design and implement context management systems for LLM applications
- Optimize token usage and context window efficiency
- Develop context compression and summarization strategies
- Implement semantic chunking and retrieval systems
- Build and maintain vector databases for context retrieval
- Create context-aware caching mechanisms
- Design conversation memory architectures
- Implement RAG (Retrieval-Augmented Generation) pipelines
- Monitor and analyze context performance metrics
- Develop context prioritization algorithms

## Required Skills
- LLM architecture understanding (attention mechanisms, context windows)
- Vector databases (Pinecone, Weaviate, Chroma, Qdrant)
- Embedding models and semantic search
- RAG implementation and optimization
- Token optimization techniques
- Memory management strategies
- Python programming and LLM frameworks
- Document chunking and preprocessing
- Caching strategies and implementations
- Performance monitoring and optimization
