# Site Reliability Engineer (SRE)

## Overview
The SRE ensures system reliability, availability, and performance through engineering practices, monitoring, and incident response while balancing feature velocity with stability.

## Key Responsibilities
- Define and maintain SLIs, SLOs, and error budgets
- Build monitoring and alerting systems
- Conduct incident response and postmortems
- Automate operational tasks
- Perform capacity planning
- Optimize system performance
- Implement chaos engineering
- Develop reliability tools
- Create runbooks and documentation
- Collaborate with development teams

## Required Skills
- Systems engineering
- Programming (Python, Go)
- Monitoring and observability
- Incident management
- Performance optimization
- Distributed systems
- Cloud infrastructure
- Automation skills
- Problem-solving
- Communication

## Reporting Structure
- Reports to: SRE Manager or Head of Infrastructure
- Direct Reports: None

## Success Metrics
- System availability (uptime)
- Mean time to detect (MTTD)
- Mean time to resolve (MTTR)
- SLO achievement
- Incident frequency
- Toil reduction
- Automation coverage
- Performance improvements
- Error budget consumption
