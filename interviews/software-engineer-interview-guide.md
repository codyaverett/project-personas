# Software Engineer Interview Guide

## Overview
Comprehensive interview structure and questions for evaluating software engineering candidates at various levels.

## Interview Structure

### Round 1: Technical Phone Screen (45-60 min)
- Introductions (5 min)
- Technical discussion (15 min)
- Coding problem (25 min)
- Questions from candidate (5 min)

### Round 2: On-site/Virtual Loop (4-5 hours)
1. Coding Interview 1 - Data Structures
2. Coding Interview 2 - Algorithms
3. System Design (Senior+)
4. Behavioral Interview
5. Team Fit Discussion

## Technical Questions by Category

### Data Structures & Algorithms

#### Easy Level
- Reverse a linked list
- Find duplicate in array
- Implement stack using queues
- Valid parentheses checker
- Binary tree traversal

#### Medium Level
- LRU Cache implementation
- Find median in data stream
- Serialize/deserialize binary tree
- Word ladder problem
- Meeting rooms scheduler

#### Hard Level
- Design autocomplete system
- Find shortest path in maze
- Implement regex matcher
- Maximum flow in graph
- Design memory allocator

### System Design Questions

#### Junior Level
- Design URL shortener
- Design parking lot system
- Design library management system

#### Mid Level
- Design Twitter timeline
- Design Uber/Lyft
- Design distributed cache

#### Senior Level
- Design YouTube/Netflix
- Design global payment system
- Design real-time collaboration tool

### Coding Best Practices
- Code organization and structure
- Error handling approaches
- Testing strategies
- Performance optimization
- Security considerations

## Behavioral Questions

### Technical Growth
- Describe a challenging bug you solved
- How do you stay updated with technology?
- Explain a complex technical concept simply
- Describe a time you had to learn quickly
- How do you approach technical debt?

### Collaboration
- Describe working with difficult teammate
- How do you handle disagreements on technical decisions?
- Tell me about mentoring someone
- How do you communicate with non-technical stakeholders?
- Describe cross-team collaboration experience

### Problem Solving
- Walk through your debugging process
- How do you prioritize tasks?
- Describe a time you failed and learned
- How do you handle ambiguous requirements?
- Tell me about an innovative solution you created

### Leadership (Senior+)
- How do you influence without authority?
- Describe leading a technical initiative
- How do you balance speed vs quality?
- Tell me about driving technical standards
- How do you handle technical debt decisions?

## Technical Assessment Rubric

### Coding Skills (1-5 scale)
1. **Problem Solving**
   - Understanding requirements
   - Breaking down problems
   - Edge case consideration

2. **Code Quality**
   - Readability and organization
   - Proper abstractions
   - Error handling

3. **Algorithms & DS**
   - Appropriate choice
   - Time/space complexity
   - Optimization ability

4. **Testing**
   - Test case design
   - Edge case coverage
   - Testing approach

5. **Communication**
   - Explaining approach
   - Asking clarifying questions
   - Receptive to feedback

### System Design (Senior+)
1. **Requirements Gathering**
   - Functional requirements
   - Non-functional requirements
   - Scale clarification

2. **High-Level Design**
   - Component identification
   - API design
   - Data flow

3. **Detailed Design**
   - Database schema
   - Algorithm choices
   - Technology selection

4. **Scalability**
   - Bottleneck identification
   - Scaling strategies
   - Performance optimization

5. **Trade-offs**
   - Cost vs performance
   - Consistency vs availability
   - Complexity vs maintainability

## Red Flags to Watch For

### Technical
- Unable to code simple problems
- No consideration of edge cases
- Lack of testing mindset
- Poor code organization
- No optimization awareness

### Behavioral
- Blaming others for failures
- No examples of collaboration
- Resistance to feedback
- Lack of learning mindset
- Poor communication skills

### Cultural
- Misaligned values
- No interest in product/business
- Lack of empathy
- Unwillingness to mentor
- No growth mindset

## Interview Best Practices

### For Interviewers
- Create comfortable environment
- Start with easier questions
- Provide hints when stuck
- Focus on thought process
- Take detailed notes
- Avoid bias
- Give candidate time to think
- Be respectful of time

### Selling the Company
- Share your experience
- Highlight growth opportunities
- Discuss technical challenges
- Explain company culture
- Show career progression
- Be honest about challenges

## Legal Compliance
- Avoid discriminatory questions
- Focus on job-related skills
- Document objective criteria
- Consistent evaluation process
- Respect confidentiality
